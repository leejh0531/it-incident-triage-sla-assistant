# IT Incident Triage & SLA Risk Assistant

## Overview

This project is a simple IT operations support system for internal incident and helpdesk tickets.

It supports ticket priority assignment, SLA risk analysis, responsible team recommendation, related FAQ search, dashboard visualization, operational insight generation, and synthetic ticket data generation.

The goal of this project is not only to visualize ticket data, but also to support operational decision-making in IT service management.

## Background

In IT operations, support teams often receive many incident and inquiry tickets.

If tickets are managed manually, it can be difficult to:

* Identify high-priority issues
* Detect SLA violation risks before the deadline
* Assign tickets to the appropriate support team
* Reuse previous troubleshooting knowledge
* Understand operational bottlenecks

This project was created as a small system development exercise to understand how IT specialists can connect business problems with technical solutions.

---

## Main Features

* Automatic ticket priority assignment based on impact and urgency
* SLA risk score calculation
* SLA status classification: Normal, Warning, High Risk, Violated
* Responsible support team recommendation
* Related FAQ search using TF-IDF
* Dashboard visualization using Streamlit
* Operational insight generation
* Processed ticket CSV export
* Synthetic IT ticket data generation

## Data

This project uses synthetic IT ticket and FAQ data created for demonstration purposes.

Main data files:

## SLA Risk Score

The SLA risk score shows how close a ticket is to violating its SLA deadline.

```text
SLA Risk Score = elapsed hours / SLA limit hours × 100
```

|        Score | Status    |
| -----------: | --------- |
|        0-50% | Normal    |
|       50-80% | Warning   |
|      80-100% | High Risk |
| 100% or more | Violated  |

---

## Priority Rule

Ticket priority is determined by combining impact and urgency.

| Impact | Urgency | Priority |
| ------ | ------- | -------- |
| High   | High    | Critical |
| High   | Medium  | High     |
| Medium | High    | High     |
| Medium | Medium  | Medium   |
| Others | Others  | Low      |

---

## FAQ Search

The FAQ search function uses TF-IDF and cosine similarity.

The system compares a ticket description with FAQ questions and answers, then returns the most relevant FAQ entries.

This function is intended to support reuse of existing troubleshooting knowledge and reduce repeated manual support work.

---

## Operational Insights

The system generates simple operational insights based on processed ticket data.

For example, it can identify:

* The most frequent incident category
* Teams with many high-risk or violated SLA tickets
* The number of unresolved tickets
* Areas that may require operational review

# IT Incident Triage & SLA Risk Assistant 한국어 설명

## 개요

이 프로젝트는 사내 IT 문의 및 장애 티켓을 관리하기 위한 간단한 IT 운영 지원 시스템입니다.

티켓의 우선순위 판정, SLA 위험도 분석, 담당팀 추천, 관련 FAQ 검색, 대시보드 시각화, 운영 개선 인사이트 생성, 가상 티켓 데이터 생성을 지원합니다.

이 프로젝트의 목적은 단순히 티켓 데이터를 시각화하는 것이 아니라, IT 서비스 관리에서 운영 담당자의 판단을 지원하는 것입니다.

---

## 배경

IT 운영 현장에서는 장애나 문의 티켓이 지속적으로 발생합니다.

이러한 티켓을 수동으로 관리하면 다음과 같은 문제가 발생할 수 있습니다.

* 우선순위가 높은 문제를 빠르게 파악하기 어렵다
* SLA 위반 위험을 마감 전에 확인하기 어렵다
* 티켓을 적절한 담당팀에 배정하기 어렵다
* 기존 해결 지식을 재활용하기 어렵다
* 운영상의 병목을 빠르게 파악하기 어렵다

이 프로젝트는 IT 스페셜리스트가 업무상의 문제를 기술적인 해결책으로 연결하는 과정을 이해하기 위해 만든 작은 시스템 개발 프로젝트입니다.

## SLA 위험도 점수

SLA 위험도 점수는 현재 티켓이 SLA 위반에 얼마나 가까운지를 나타내는 지표입니다.

```text
SLA Risk Score = 경과 시간 / SLA 기준 시간 × 100
```

|      점수 | 상태        | 의미                  |
| ------: | --------- | ------------------- |
|   0-50% | Normal    | 아직 여유가 있는 상태        |
|  50-80% | Warning   | 주의가 필요한 상태          |
| 80-100% | High Risk | 곧 SLA 위반 가능성이 높은 상태 |
| 100% 이상 | Violated  | SLA 기준을 초과한 상태      |

---

## 우선순위 판정 규칙

티켓의 우선순위는 영향도와 긴급도를 조합하여 판정합니다.

| 영향도    | 긴급도    | 우선순위     |
| ------ | ------ | -------- |
| High   | High   | Critical |
| High   | Medium | High     |
| Medium | High   | High     |
| Medium | Medium | Medium   |
| 그 외    | 그 외    | Low      |

## FAQ 검색

FAQ 검색 기능은 TF-IDF와 코사인 유사도를 사용합니다.

시스템은 티켓 설명과 FAQ의 질문 및 답변을 비교하여 가장 관련성이 높은 FAQ 항목을 반환합니다.

이 기능은 기존 문제 해결 지식을 재활용하고, 반복적인 수동 대응 업무를 줄이는 것을 목적으로 합니다.
