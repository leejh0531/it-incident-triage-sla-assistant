# Requirements

## 1. Functional Requirements

- The system shall load IT ticket data from CSV.
- The system shall assign ticket priority based on impact and urgency.
- The system shall calculate SLA risk score.
- The system shall classify SLA status as Normal, Warning, High Risk, or Violated.
- The system shall recommend a responsible support team.
- The system shall search related FAQs using ticket descriptions.
- The system shall display dashboard metrics and charts.
- The system shall export processed ticket data as CSV.

## 2. Non-functional Requirements

- The system should run locally.
- The logic should be simple and explainable.
- The interface should be easy to use.
- The system should not use real company or personal data.
- The project should be easy to extend in the future.
